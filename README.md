# ⚡ EcoElectricity

> 🔗 [github.com/Jasurios/EcoElectricity](https://github.com/Jasurios/EcoElectricity)

🌍 Интерактивная 3D-карта электростанций мира на спутниковых снимках с рельефом местности, зонами загрязнения, кинематографичным облётом камеры и подробной статистикой по странам.

## ✨ Возможности

- 🛰️ **3D спутниковая карта** на тайлах Esri World Imagery с объёмным рельефом (AWS Terrarium DEM)
- 🏭 **150+ электростанций** по всему миру: ГЭС, АЭС, угольные, газовые, солнечные и ветровые станции — почти все с фотографиями
- 📦 **Данные в JSON**, загружаются на клиенте простым `fetch` — без какого-либо backend-сервера
- 💧⚛️🔥☀️🌬️ **Маркеры с уникальными иконками** под каждый тип станции (капля, атом, дымоход, пламя, солнце, турбина)
- ⚠️ **Зоны воздействия на карте**: загрязнение вокруг угольных/газовых станций, шум вокруг ветропарков
- 🚀 **Облёт камеры**: кнопка «Показать» на любой станции — камера подлетает, наклоняется на 70° и кружит вокруг объекта; «Стоп» возвращает камеру туда, где она была
- 🔍 **Поиск станций** по названию и стране (EN/RU) с выпадающими подсказками — можно сразу запустить облёт прямо из поиска
- 📊 **Статистика по странам**: общая мощность, топ стран, donut-диаграмма по типам генерации
- 🗂️ **Боковая панель** с фото, описанием, мощностью, годовой выработкой, страной, рекой и количеством агрегатов
- 🌐 **Переключение языка** EN / RU для всего интерфейса
- ⚡ **Оптимизация рендера**: станции появляются по мере приближения камеры — лёгкая карта даже с полутора сотней маркеров

## 🧱 Стек

- ⚛️ React 18 + Vite
- 🗺️ MapLibre GL JS (рендер карты, 3D-рельеф, кастомные слои, облёт камеры)
- 📄 Статический JSON как «база данных» станций — никакого backend-сервера
- 🎨 CSS Modules

## 🚀 Запуск

```bash
git clone https://github.com/Jasurios/EcoElectricity.git
cd EcoElectricity
npm install
npm run dev
```

Откройте адрес, который выведет Vite (по умолчанию `http://localhost:5173`).

## 📦 Сборка

```bash
npm run build    # production-сборка в dist/
npm run preview  # просмотр собранной версии локально
```

## 🗄️ База данных станций

Данные о станциях хранятся в файле `public/power-plants.json`. Приложение просто
делает `fetch('/power-plants.json')` при загрузке страницы — никакого backend-сервера
не требуется.

Каждая станция содержит: координаты (`lng`/`lat`), тип, мощность (`capacity`, МВт),
страну, реку (если есть), статус (`active` / `construction` / `decommissioned`), год
постройки, годовую выработку, число агрегатов, ссылку на фото (`img`) и описание на
английском и русском.

Чтобы добавить/изменить станции:

1. Отредактируйте массив в `scripts/plants-data.mjs`
2. Пересоберите JSON: `npm run data:build`
3. Закоммитьте обновлённый `public/power-plants.json`

```bash
npm run data:build   # пересобрать public/power-plants.json из scripts/plants-data.mjs
```

📌 `scripts/plants-data.mjs` сделан отдельным источником данных специально, чтобы при
необходимости его легко перенести в настоящую БД (SQLite/Postgres/etc.) в будущем —
сейчас он используется только для генерации JSON.

Картинки для станций без готового фото подбираются автоматически через Wikipedia API:

```bash
node scripts/fetch-images.mjs   # дозаполнить scripts/fetched-images.json фото станций
```

## 📁 Структура проекта

```
src/
  DashboardPage.jsx        главный экран: карта, маркеры, зоны, поиск, облёт камеры, боковая панель
  StatsPanel.jsx            модальное окно со статистикой по странам и типам генерации
  constants.js              общие константы (цвета/иконки типов станций)
  Dashboard.module.css      стили главного экрана
  Stats.module.css          стили панели статистики
  App.jsx                   точка входа в приложение
  main.jsx                  монтирование React-приложения
  index.css                 глобальные стили
scripts/
  plants-data.mjs           исходный массив всех станций (имя, координаты, мощность, фото, описание)
  build-json.mjs            собирает scripts/plants-data.mjs в public/power-plants.json
  fetch-images.mjs          подбирает фото станций без картинки через Wikipedia API
public/
  power-plants.json         готовые данные станций, отдаются как статический файл
```
